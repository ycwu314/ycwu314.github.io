多级爬取

爬取思路：

  • 首页解析音乐类别列表（genre 列表）
  • 分页爬取每个音乐类别的歌曲列表（play list 列表）
  • 爬取每个歌曲播放页面的元数据（play music 列表）

整个爬取流程经历：

首页 -> 音乐分类 -> 歌曲列表 -> 歌曲页面

最终要解析元数据的页面是歌曲页面。但是要经历前面 3 个页面，才能遍历全量的歌曲页面地址。
scrapy 的 start_requests 方法用于初始化爬取网页 url。以前写简单的页面爬取，直接在 parse 方法做最终页面的解析即可。难道要在 parse 方法塞几个页面的解析逻辑？
Off course not! scrapy 的 Request 支持 callback。不同页面的设置不同回调函数解析即可。

xpath 解析

解析音乐文件路径，发现 url 不是写在某个属性，而是在 script 里面拼接。

首先要定位这个 script，发现 xpath 表达式在页面工具和 scrapy 解析有兼容性问题，单纯计算 script index 定位的结果不一致😂。
那就用文本匹配。涉及 2 个 xpath 函数：

  • contains()：匹配一个属性值中包含的字符串
  • text()：显示文本信息

定位 script 标签，再转为文本做正则匹配即可。

同样方式得到 fileHost 等属性。
